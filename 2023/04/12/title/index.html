智能Agent

Agent和环境

Agent是可以通过传感器感知其环境并通过执行器在该环境上作用的任何事物。

  • Agent 函数，其中为感知序列，为行动；

  • Agent程序是的具体实现；

  • 体系结构是指Agent程序要在某个具备物理传感器和执行器的计算装置上运行；

Agent=体系结构加程序。

理性

理性Agent的定义：对每一个可能的感知序列，根据已知的感知序列提供的证据和 Agent 具有的先验知识，理性 Agent 应该选择能够使其性能度量最大化的行动。（理性≠完美）

判断理性依赖于四个方面：

  • 性能度量
  • 先验知识
  • 行动
  • 感知序列

环境

要构建一个理性Agent，首先必须指定任务环境，也就是PEAS (Performance， Environment， Actuators， Sensors)。

Performance， Environment， Actuators， Sensors

   性能度量   环境    执行器 传感器

环境类型

  1. 完全可观察的 vs. 部分可观察的：是否能获取环境的完整状态。
  2. 单Agent vs. 多Agent：一个Agent的行为是否寻求让依赖于另一个Agent的行为性能度量最大化。
  3. 确定的 vs. 随机的：环境的下一状态是否完全取决于当前状态和Agent执行的动作。（如果环境是部分可观察的，那么它可能表现为随机的）
  4. 片段式的 vs. 延续式的：Agent的经历被分成多个相互无依赖原子片段or当前决策会影响所有未来的决策。
  5. 静态的 vs. 动态的：如果环境在Agent计算时会变化，那么是动态的，反之是静态的。如果环境本身不随时间变化，但Agent性能评价随时间动态变化，那么是半动态的。
  6. 离散的 vs. 连续的
  7. 已知的 vs. 未知的

现实世界始终是部分可观察的、随机的、延续的、动态的、连续的和多Agent的。

Agent 类型

简单反射 Agent

动作完全取决于条件-行为规则，也称为无内存或无状态。仅在完全可观察的环境下有效。如果环境是部分可观察的，则通常陷入无限循环。

基于模型的 Agent

让Agent跟踪记录现在看不到的那部分世界。维持内部状态来对世界进行建模。

基于目标的 Agent

使用目标来决定其执行的操作。

基于效用的 Agent

最大化期望效用。（效用函数：Agent的快乐度）

学习 Agent

从奖励（或惩罚）中学习，学习技术形成了机器学习领域。

所有这些Agent均可以转换为学习Agent

无信息搜索

问题类型 问题形式化

确定的，完全可观察的 ⟹ 单状态问题（Agent确切知道它将处于哪个状态；问题的解是一个序列。）

不可观察的 ⟹ 一致性问题（Agent可能不知道它在哪里；问题的解（如果有）是一个序列。）

不确定性的和/或部分可观察的 ⟹ 偶然性问题

未知状态空间 ⟹ 探索问题（在线问题）

问题形式化：初始状态，行动，转移模型，目标测试，路径消耗

搜索树和图

树搜索算法的基本思路：离线算法，通过生成已经探索过的状态的继任者模拟搜索状态空间。

树搜索的缺点：不能检测到重复状态，会导致冗余路径。所以图搜索会有一个closed表（探索集）来记录已经扩展过的结点。

图搜索算法：树搜索算法 + 探索集。图搜索算法可以直接在状态空间中生成一棵树。图搜索可以比树搜索成倍提高效率。

搜索策略的评估：完备性，最优性，时间复杂度，空间复杂度。

  • 完备性：是否总能找到解，当问题有解时。
  • 最优性：是否总能找到最优解。
  • 时、空复杂度：略

时空复杂度的度量：最大分支因子（任何结点的最多后继数），最优解深度，路径最大长度 。

深度优先搜索（DFS）

详细过程略

  • 完备性：否 （需修改以避免在路径上重复状态，才能在有限空间上完备）
  • 最优性：否
  • 时空复杂度：时间复杂度，空间复杂度为线性空间。

深度受限搜索（DLS）

为什么会有这种算法：如果状态空间中路径的最大长度，则DFS永不终止。

深度受限搜索DLS = 深度限制为的深度优先搜索DFS，即深度为的结点没有后继结点。

  • 完备性：视深度限制和最优解深度
  • 最优性：通常不是最优的，无论是多少
  • 时空复杂度：与DFS相似

迭代加深的深度优先搜索（IDS）

算法过程：逐渐增大深度限制，迭代调用深度受限搜索DLS。

浪费冗余：通常绝大多数的节点都在底层，所以上层节点生成多次影响不是很大

  • 完备性：类似BFS
  • 最优性：类似DFS
  • 时空复杂度：类似BFS

IDS结合了BFS的时间优势和DFS的空间优势。

双向搜索（Bidirectional Search）

运行两个搜索，也就是一个从初始状态向前搜索，同时另一个从目标状态向后搜索，希望它们在中间某点相遇，此时搜索终止。

  • 完备性：如果有限，且两端都使用BFS

  • 最优性：大部分情况下不是最优（所有路径消耗相同时最优）

A*搜索

在讲A*搜索之前，我们必须先清楚可采纳和一致两个概念（看到一位学长的博客说没整理到这个然后考试被坑了）

  • 可采纳性（admissible）：，是从结点到目标结点的真实代价。同时要求，因此对任意目标，。(我的理解大概就是你选取的方法对于评估你的代价估计值是行之有效的，你选的总不可能比你真实值还要大吧)
  • 一致性（consistent）：如果对于每个结点和通过任一行动生成的的每个后继结点，从结点到达目标的估计代价不大于从到的单步代价与从到达目标的估计代价之和：，也就是说在任何路径上都是非递减的。实际上是三角不等式

结论：如果是可采纳的，则树搜索的A算法是最优的。如果是可采纳且一致的，则图搜索的A算法是最优的。

如何寻找可采纳启发式？（松弛问题）

个人理解：因为如果不是一致的，那么树搜索会利用可往回搜索的特性来找到最优解，而图搜索找不到。

回归正题，A*搜索的思想是避免扩展代价已经很高的路径，结合了一致代价搜索和贪婪最佳优先搜索的优点。

局部搜索算法

局部搜索算法从单个当前结点（而不是多条路径）出发，通常试图通过只移动到它的邻近状态从而提升它。

爬山法

不断向值增加的方向持续移动，不考虑与当前状态不相邻的状态。

模拟退火搜索

爬山法不会向值比当前结点低（或代价高的）方向搜索，因此是不完备的。随机游走是完备的，但效率极低。

什么是退火？将固体加温至充分高，再让其徐徐冷却，加温时，固体内部粒子随温升变为无序状，内能增大，而徐徐冷却时粒子渐趋有序，在每个温度都达到平衡态，最后在常温时达到基态，内能减为最小。

模拟退火算法从某一较高初温出发，伴随温度参数的不断下降,结合概率突跳特性在解空间中随机寻找目标函数的全局最优解，即在局部最优解能概率性地跳出并最终趋于全局最优。

例子：高地不平的平面上有个乒乓球想掉到最深的裂缝中

  • 如果只允许乒乓球滚动，很可能停留在局部极小点。（乒乓球爬山）
  • 如果平面晃动，可以是乒乓球弹出局部极小点。（乒乓球随机游走）
  • 技巧是晃动幅度要足够大使乒乓球弹出局部极小点，但不能太大，把它从全局最小点弹出来。

模拟退火的解决办法：开始使劲摇晃，然后慢慢降低强度。

局部束搜索

从个随机生成的状态开始。每一步生成全部个状态的所有后继状态。如果有一个是目标状态，则算法终止。否则，选择所有后继状态中的前个最佳状态，重复上述过程。

随机束搜索：随机选择个后继状态，偏向好的后继状态。类似于自然选择，状态（生物体）根据值（适应度）产生后继（后代子孙）。

是不是有点遗传算法那味儿了。

遗传算法

随机束搜索的一个变形，通过把两个父状态结合生成后继，而不是通过修改单一状态进行。

遗传算法 随机束搜索 从状态对生成后继。（选择、杂交、变异）

使用不确定性动作的搜索

如果环境是部分可观察的或是不确定的，无法预知未来感知信息，Agent的未来行动依赖于未来感知信息。

所以问题的解不是一个序列，而是一个应急规划（也称作策略），应急规划描述了根据接收到的感知信息来决定行动。

使用部分可观察信息的搜索

部分可观察问题下Agent的感知不足以精确描述状态解决部分可观察问题的关键概念：信念状态（belief states）

信念状态：在给定行动序列和感知信息的情况下用来表达Agent对它当前所在的物理世界的信念。

无观察信息的搜索

求解无观察信息的问题，是在信念状态空间中而不是实际状态空间中搜索，解总是一个行动序列。在信念状态空间中，问题是完全可观察的，因为Agent总是知道自己的信念状态。

有观察信息的搜索

有观察信息指的是Agent可以感知到部分信息，但不是完整信息。多了一个感知函数PERCEPT()。

求解部分可观察环境中的问题
